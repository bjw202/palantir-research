## 슬라이드 3: "우리 공장에 데이터가 없는 것이 아닙니다"

**레이아웃:** 상단에 7개 시스템 아이콘 가로 배치(각 아이콘 사이에 빨간 X), 하단에 핵심 메시지 대형 텍스트
**배경:** 밝은 회색(#F5F5F5) 배경
**애니메이션:** 7개 아이콘 동시 등장 → 아이콘 사이 빨간 X 순차 등장(끊어짐 강조) → 하단 메시지 페이드인

### 핵심 메시지
> 데이터는 넘친다. 문제는 연결되지 않는다는 것이다.

### 콘텐츠

**[상단 70% — 7개 시스템 가로 배치]**

```
 [ERP]  ✗  [MES]  ✗  [SCADA]  ✗  [QMS]  ✗  [WMS]  ✗  [CMMS]  ✗  [PLM]
  📊        🏭        📡         🔍        📦        🔧        📐
```

- 각 아이콘 아래에 시스템 약칭과 "데이터 있음" 초록 체크 표시
- 아이콘 사이의 빨간 X는 크고 굵게 (연결 단절 강조)

**[하단 30% — 핵심 메시지]**

### 7개 시스템, 7개의 섬
> **데이터는 넘칩니다. 연결되지 않을 뿐입니다.**

### 발표자 노트
우리 공장에는 ERP부터 PLM까지 7개 이상의 시스템이 운영되고 있습니다. 각 시스템은 충실히 데이터를 기록하고 있습니다. 문제는 이 시스템들이 서로 대화하지 못한다는 겁니다. 10~20년에 걸쳐 다른 시기에, 다른 벤더 제품으로, 다른 부서가 도입했기 때문입니다.

## 슬라이드 4: "같은 설비, 세 개의 이름"

**레이아웃:** 중앙에 CNC 설비 일러스트, 좌/상/우 세 방향에서 3개 시스템이 각자의 이름표를 붙이는 구도
**배경:** 흰색 배경
**애니메이션:** 중앙 설비 등장 → 세 방향에서 이름표가 날아와 붙음(각 0.4초 간격) → 하단에 "?" 강조 등장

### 핵심 메시지
> 같은 설비를 세 시스템이 서로 다른 이름으로 부르며, 누구도 이것이 같은 설비인지 모른다.

### 콘텐츠

**[중앙 — CNC 설비 일러스트]**

```
                    ERP
                 "EQ-001-A"
                     │
                     ▼
        MES ──→  [CNC 설비]  ←── CMMS
   "MACHINE_A_LINE1"   실체    "ASSET-00142"
```

**[하단 강조 텍스트]**

> **하나의 설비, 세 개의 이름, 연결 불가**
>
> 마스터 데이터 관리(MDM)가 없으면 시스템 간 연결 자체가 불가능합니다.

### 발표자 노트
구체적 예를 보겠습니다. 우리 공장의 CNC 설비 하나가 ERP에서는 "EQ-001-A", MES에서는 "MACHINE_A_LINE1", CMMS에서는 "ASSET-00142"로 불립니다. 세 시스템 모두 같은 설비를 가리키지만, 서로 이것이 같은 설비라는 것을 모릅니다. 이것이 데이터 사일로의 가장 기본적인 문제입니다.

## 슬라이드 5: "불량이 발생하면 — 현장의 현실"

**레이아웃:** 좌→우 타임라인, 5개 단계를 시간축 위에 배치, 각 단계에 시스템 아이콘 + 소요 시간
**배경:** 흰색 배경, 타임라인은 빨간→주황 그라데이션(위기감 표현)
**애니메이션:** 타임라인이 좌→우로 진행하며 각 단계가 순차 등장, 마지막에 총 소요 시간 강조 등장

### 핵심 메시지
> 불량 원인을 찾는 데 최소 6시간에서 최대 3일이 걸린다.

### 콘텐츠

**[타임라인 — 좌→우 진행]**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━→ 시간

  [QMS 로그인]     [MES 조회]      [SCADA 추출]     [CMMS 확인]      [엑셀 병합+분석]
  불량 이력 조회    공정 이력 조회    센서 데이터 추출   정비 이력 확인     수작업 교차 분석
    30분             1시간           1~2시간           30분            2~4시간
```

**[우측 끝 — 강조 박스]**

```
┌──────────────┐
│  총 소요 시간   │
│  6시간 ~ 3일   │
└──────────────┘
```

### 발표자 노트
불량이 발생하면 엔지니어는 QMS에서 불량 이력을 확인하고, MES에서 Lot 공정 이력을 조회하고, SCADA에서 센서 데이터를 추출하고, CMMS에서 정비 이력을 확인합니다. 4개 시스템에 각각 로그인해서 엑셀로 데이터를 병합하는 데 최소 6시간, 복잡한 경우 3일까지 걸립니다.

## 슬라이드 6: "그 사이, 불량은 계속 만들어집니다"

**레이아웃:** 좌우 분할 — 좌측에 분석하는 엔지니어(시계 애니메이션), 우측에 컨베이어 벨트에서 불량품이 계속 나오는 일러스트
**배경:** 좌측 밝은 톤 / 우측 빨간 경고 톤으로 분할
**애니메이션:** 좌측 시계 회전 + 우측 불량품 하나씩 추가(총 8개까지)

### 핵심 메시지
> 분석하는 동안 동일 불량이 계속 생산된다. 5시간 분석 중 불량 Lot 8개 추가.

### 콘텐츠

**[좌측 50% — 분석 중]**

```
     ┌─────────────┐
     │  엔지니어     │
     │  [분석 중...] │
     │              │
     │   ⏱ 5시간    │
     │   경과        │
     └─────────────┘

     4개 시스템 로그인
     엑셀 병합 중...
     원인 가설 검증 중...
```

**[우측 50% — 생산 라인]**

```
     컨베이어 벨트 ━━━━━━━━→

     [불량] [불량] [불량] [불량]
     [불량] [불량] [불량] [불량]

     ⚠ 불량 Lot 8개 추가 생산
```

**[하단 전체 — 강조 메시지]**

> **분석 5시간 동안, 불량 Lot 8개가 추가로 생산됩니다.**

### 발표자 노트
여기서 정말 심각한 문제가 드러납니다. 엔지니어가 원인을 분석하는 5시간 동안, 공장은 멈추지 않습니다. 원인을 모르니까 멈출 수도 없고, 전체 라인을 세우기엔 생산 손실이 너무 큽니다. 그 사이 동일한 조건에서 불량 Lot 8개가 추가로 생산됩니다. 원인을 빨리 찾는 것이 곧 비용 절감입니다.

## 슬라이드 7: "야간에는 더 심각합니다"

**레이아웃:** 좌우 분할 — 좌측(낮/밝은 톤)과 우측(밤/어두운 톤) 대비
**배경:** 좌측 밝은 파란색, 우측 진한 남색(#0A1628)
**애니메이션:** 좌측 먼저 등장 → 우측 등장 → 우측의 지연 시간이 빨간색으로 강조

### 핵심 메시지
> 야간/주말에는 의사결정 공백으로 대응이 1~2시간 이상 추가 지연된다.

### 콘텐츠

**[좌측 50% — 주간]**

```
       ☀ 주간 (08:00~17:00)
    ─────────────────────
    공정 엔지니어    3명
    품질 엔지니어    2명
    설비 엔지니어    2명
    ─────────────────────
    → 즉시 다부서 협의 가능
    → 대응 시간: 2시간 이내
```

**[우측 50% — 야간]**

```
       🌙 야간 (22:00~06:00)
    ─────────────────────
    교대 운전원      2명
    당직 설비 기사    1명
    ─────────────────────
    → 전화 에스컬레이션 필요
    → 전문가 수면 중 (응답 30분~1시간)
    → 원격 접속 미비 시 출근 (+1~2시간)

    ⚠ 대응 시간: 최대 7시간+
```

### 발표자 노트
공장은 24시간 돌아가지만, 의사결정 구조는 주간 근무 기준입니다. 주간에는 전문 엔지니어 7명이 즉시 대응하지만, 야간에는 운전원 2명과 당직 기사 1명뿐입니다. 복합적 문제가 발생하면 전화로 에스컬레이션해야 하고, 전문가가 수면 중이라 응답까지 30분에서 1시간, 출근까지 추가 1~2시간이 걸립니다.

## 슬라이드 8: "현재 비용 — 매년 얼마를 잃고 있는가"

**레이아웃:** 좌측 70%에 누적 막대 그래프, 우측 30%에 합계 강조 박스
**배경:** 흰색 배경
**애니메이션:** 막대가 아래→위로 순차 쌓임 → 합계 숫자 카운트업 애니메이션 → 매출 비율 강조

### 핵심 메시지
> 매년 290~360억 원, 매출의 29~36%가 데이터 단절로 인한 비효율로 손실된다.

### 콘텐츠

**[좌측 70% — 누적 막대 그래프]**

| 비용 항목 | 금액 | 색상 |
|-----------|------|------|
| 품질 비용 (COPQ) | ₩150~200억 | 빨간색 |
| 비계획 다운타임 | ₩110억 | 주황색 |
| 정비 비효율 | ₩20~30억 | 노란색 |
| 의사결정 지연 | ₩10~20억 | 회색 |

**[우측 30% — 합계 강조]**

```
┌──────────────────┐
│                  │
│  ₩290~360억/년   │
│                  │
│  매출 1,000억 기준 │
│  매출의 29~36%    │
│                  │
└──────────────────┘
```

**[하단 — 출처]**

> 출처: Siemens 2024, IISE, Parsec | 매출 1,000억 원 규모 제조사 기준

### 발표자 노트
이 문제들을 비용으로 환산하면 충격적입니다. 품질 비용만 연 150~200억 원, 비계획 다운타임 110억 원, 정비 비효율과 의사결정 지연까지 합하면 연간 290~360억 원입니다. 매출 1,000억 원 기준으로 29~36%에 해당합니다. Siemens 2024년 보고서에 따르면 세계 500대 기업의 비계획 다운타임이 전 세계 1.4조 달러 규모입니다.

## 슬라이드 9: "기존 접근이 실패한 이유"

**레이아웃:** 4개 원형 아이콘을 가로 배치, 각각에 O/X 표시, 하단에 공통 메시지
**배경:** 밝은 회색(#F5F5F5) 배경
**애니메이션:** 4개 원형이 동시 등장 → 각각의 O/X가 순차 등장 → 하단 메시지 강조 등장

### 핵심 메시지
> BI, ML, 데이터 레이크, RPA를 모두 도입해도 "연결하는 층"이 없으면 통합 흐름은 만들어지지 않는다.

### 콘텐츠

**[4개 원형 — 가로 배치]**

```
   BI 대시보드        ML 모델          데이터 레이크         RPA
  ┌──────────┐   ┌──────────┐    ┌──────────┐     ┌──────────┐
  │          │   │          │    │          │     │          │
  │  What    │   │  예측     │    │  저장     │     │  반복     │
  │  만 답함  │   │  만 함    │    │  만 함    │     │  만 함    │
  │          │   │          │    │          │     │          │
  └──────────┘   └──────────┘    └──────────┘     └──────────┘
  ○ 현황 시각화   ○ 고장 확률 예측  ○ 데이터 통합     ○ 반복 자동화
  ✗ Why 답 못함  ✗ 실행 연결 안됨  ✗ 의미/관계 없음   ✗ 판단 불가
          (유럽 조사 기준 11%)  (Gartner 85%)
```

**[하단 — 강조 메시지]**

> **네 가지를 모두 도입해도, "데이터를 이해하고 → 관계를 추론하고 → 판단하고 → 실행하는" 통합 흐름은 만들어지지 않습니다.**
>
> 각각의 사이를 연결하는 층이 없기 때문입니다.

### 발표자 노트
그렇다면 기존에 시도한 접근들은 왜 부족했을까요? BI는 현황만 보여주고 왜 그런지는 모릅니다. ML은 예측하지만 실행으로 이어지지 않아 유럽 기업 조사 기준 성공률이 11%에 불과합니다(Springer, 2024). 데이터 레이크는 데이터를 모았지만 의미 없는 늪이 되어 Gartner에 따르면 85%가 실패합니다. RPA는 반복 작업은 하지만 판단을 못 합니다. 이 네 가지 사이를 연결하는 층이 빠져 있습니다.

## 슬라이드 10: "문제의 본질"

**레이아웃:** 임팩트 슬라이드 — 큰 텍스트 중앙 배치, 하단에 3개 키워드
**배경:** 진한 남색(#0A1628) 배경, 흰색 텍스트
**애니메이션:** 핵심 메시지 타이핑 효과로 등장 → 3개 키워드 동시 팝업

### 핵심 메시지
> 데이터에 의미와 관계와 행동이 없다 — 이것이 모든 문제의 근인이다.

### 콘텐츠

**[중앙 — 대형 텍스트]**

# "데이터에 의미와 관계와 행동이 없다."

**[하단 — 3개 키워드 가로 배치]**

```
    ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
    │  Semantics  │     │Relationship │     │   Action    │
    │   의미       │  +  │   관계       │  +  │   행동      │
    │             │     │             │     │             │
    │ 같은 설비를   │     │ Lot→Machine  │     │ 감지→분석    │
    │ 같은 이름으로 │     │ →Tool→Recipe │     │ →판단→실행   │
    └─────────────┘     └─────────────┘     └─────────────┘
```

**이 세 가지가 합쳐진 것 = 온톨로지**

### 발표자 노트
지금까지 살펴본 모든 문제를 관통하는 하나의 근본 원인이 있습니다. 데이터에 의미, 관계, 행동이 없다는 것입니다. 같은 설비를 같은 이름으로 부르는 공유된 의미, Lot에서 Machine으로 Tool로 Recipe로 이어지는 관계, 그리고 감지에서 분석, 판단, 실행까지 이어지는 행동. 이 세 가지를 하나로 통합한 것이 바로 온톨로지입니다.
